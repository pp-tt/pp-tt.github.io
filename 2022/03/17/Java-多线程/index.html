Java 多线程

Posted on 2022-03-17 In Java

对 Java 多线程问题简单梳理总结。

1 相关概念

1.1 并行和并发

  • 并行：同一时刻，有多个指令在多个CPU上同时执行
  • 并发：同一时刻，有多个指令在单个CPU上交替之行

1.2 进程和线程

  • 进程： 是正在运行的软件
    • 独立性
    • 动态性
    • 并发性
  • 线程：是进程中单个顺序控制流，是一条执行路径

1.3 多线程的实现方案

  • 继承 Thread 类的方式进行实现
  • 实现 Runnable 接口的方式进行实现
  • 利用 Callable 和 Future 接口方式实现

2.2 相关问题

  • 为什么要重写 run() 方法？

    因为多线程在开启之后，实质上执行的就是 run() 方法内的内容，所以要重写 run() 方法，将其内包括的内容换成我们需要执行的内容。

  • run() 方法和 start() 方法的区别？

    run() 方法相当于普通的调用方法，即创建实例，然后调用 run() 方法，相当于直接调用；start() 是启动线程，然后由 JVM 来调用该线程的 run() 方法。

5 三种多线程方式区别

方式 优点 缺点
Thread 实现简单，可以直接调用 Thread 类中的其他方法 扩展性差，不能再继承其他的类
Runnable 扩展性好 实现较复杂，需要实现重写 Runnable 中的 run() 方法然后作为构造方法参数传递 Thread
Callable 扩展性好，有返回值 相比 Runnable 实现更复杂，先作为 FutureTask 的构造参数，然后作为 Thread 的构造参数

7 线程安全问题

7.1 同步代码块

  • 同步技术原理

    使用了一个锁对象， 这个锁对象叫同步锁，也叫对象锁，也叫对象监视器。假设三个线程 t0、t1、t2 一起抢夺CPU的执行权，说抢到了谁执行 run 方法， t0 抢到了，执行 run 方法，遇到 synchronized 代码块，这是 t0 会检查 synchronized 代码块是否有锁对象，发现有，就会获取到锁对象进入到同步中执行。t1 抢到 CPU 执行权时也会判断是否有锁对象，这时锁对象在 t0 那，发现没有锁对象，t1会进入阻塞状态，一直等待 t0 将锁对象归还给同步代码块，t1 才能获取锁对象进入到同步中执行。

  • 同步技术缺点

    程序频繁的判断锁、获取锁、释放锁，程序的效率会降低。

8 生产者和消费者

8.1 等待和唤醒的方法

为了体现生产和消费过程中的等待和唤醒，Java 的 Object 类提供以下方法

方法名 说明
void wait() 导致当前线程等待，直到另一个线程调用该对象的 notify() 方法或 notifyAll() 方法
void notify() 唤醒正在等待对象监视器的单个线程（随机）
void notifyAll() 唤醒正在等待对象监视器的所有线程

8.2 阻塞队列

  • BlockingQueue 核心方法：
    1. put(anObject): 将参数放入队列，如果放不进去会阻塞
    2. take(): 取出第一个数据，取不到会阻塞
  • 常见 BlockingQueue:
    1. ArrayBlockingQueue: 底层是数组，有界
    2. LinkedBlockingQueue: 底层是链表，无界，但不是真正的无界，最大为int的最大值

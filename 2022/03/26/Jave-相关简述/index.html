Jave EE 相关简述

Posted on 2022-03-26 Edited on 2022-03-28 In Java

Java EE (Java Enterprise Edition) 企业版，由 SUN 公司领导、各个厂家共同制定并得到广泛认可的工业标准。Java EE 规范是很多 Java 开发技术的总称，一共包括了 13 个技术规范。包括：JDBC, JNDI, EJB, RMI, IDL/CORBA, JSP, Servlet, XML, JMS, JTA, JTS, JavaMail, JAF。

系统结构

CS 结构：

(Client Server) 客户端 + 服务器

客户端安装 APP，客户端可以是 PC 端，也可以是移动端。应用数据和交互等都部署在服务器上。

BS 结构:

(Browser Server)浏览器 + 服务器

基本使用

启动

  1. Linux / MacOS 启动文件：startup.sh
  2. Windows 启动文件：startup.bat

停止

  1. Linux / MacOS 停止文件：shutdown.sh
  2. Windows 停止文件：shutdown.bat

部署自己的项目

  1. 在 webapps 目录下创建一个文件夹
  2. 将资源放到该文件夹里
  3. 启动 tomcat，输入正确路径

  1. Run -> Edit Configurations

项目组成详解

  • src：存放源代码
  • web：存放项目相关资源的（html、css、js、jsp、图片等)
  • WEB-INF：存放相关配置的（web.xml等）

tomcat 配置文件

路径 conf/server.xml ，可以修改端口号。

HTTP 协议

介绍

HTTP(Hyper Text Transfer Protocol): 超文本传输协议，基于 TCP/IP 协议。

HTTP 协议请求的组成部分

  1. 请求的组成部分

    • 请求行 ：请求提交的方式，版本号
    • 请求头 ：
    • 请求空行 ： 用于区分请求头和请求体
    • 请求体 ：只有 POST 方式才有请求体， 用于显示提交参数

  2. 请求的方式

    • GET
    • POST

    注意：只有 POST 请求方式才有请求体。

HTTP 协议相应的组成部分

  • 响应行 ：请求提交的方式版本号，状态码 状态描述

    • 常见状态码：
      • 200：正常
      • 302/307： 重定向，两次请求，地址栏发生变化
      • 304：请求资源未发生变化，使用缓存
      • 404：请求资源未找到
      • 500：服务器错误
